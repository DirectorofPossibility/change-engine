Consultant Recommendations: Community Exchange

1. The Archetype System

Your platform already has Centers (Learning, Action, Resource, Accountability) and Pathways (7 themes). What's missing is the human entry point — who is the person arriving, and what mode are they in? Archetypes bridge the gap between "what the platform organizes" and "how people actually show up."

Archetypes are NOT user accounts or profiles. They're navigational lenses — a way someone self-selects their intent in this moment, which then curates the experience.

🌱
The Seeker
Looking for resources, services, or answers for themselves or a loved one
Maps to: Resource Center
📖
The Learner
Wants to understand an issue, explore research, learn how systems work
Maps to: Learning Center
🤝
The Builder
Ready to volunteer, organize, contribute, or create something in the community
Maps to: Action Center
🔍
The Watchdog
Tracking decisions, policies, elected officials, and accountability
Maps to: Accountability Center
🏢
The Partner
An organization, funder, or institution looking to connect and collaborate
Maps to: Partner Portal
🌍
The Explorer
Just browsing, curious, wants to see what's happening across Houston
Maps to: Compass / Pathways

2. Why Archetypes in the Left Nav?

The left navigation becomes a persistent wayfinding rail that anchors the user's experience across every page. It replaces the current top-nav-only approach with a always-visible context panel.

Current State

Top nav has: Pathways, Services, Officials, Policies, Elections, Library. That's entity-type navigation — it assumes the user already knows what type of thing they want. Most don't.

Proposed State

Left nav has: Archetypes (intent-first), then Pathways (theme-second), then entity shortcuts. The top nav simplifies to search + language + zip. The user's "mode" persists across pages.

3. Unified Template Strategy

Every detail page should follow the same three-column rhythm:

Left Nav (persistent context: archetype + pathways + shortcuts) + Main Content (adapts per content type) + Wayfinder Sidebar (universal 3-tier: Understand, Get Involved, Go Deeper)

What stays consistent across ALL 8 page types:

  • Left nav with archetypes + pathways — always visible on desktop
  • Breadcrumb trail showing location in the knowledge graph
  • Compact hero: pathway dot + serif title + italic description + colored rule
  • Wayfinder sidebar with org anchors, focus dots, 3 collapsible tiers
  • "Suggest an edit" + "View source" CTAs at bottom of main content
  • Mobile: left nav becomes hamburger drawer, wayfinder collapses below main

What adapts per content type:

  • Content: hero image + photo strip + article body + inline map
  • Official: circular photo + contact grid + district map + legislation list
  • Service: cost/free badge + contact grid + location map + related services
  • Organization: logo + photo strip + stats grid + services list + area map
  • Policy: status badges + timeline + impact highlight + decision makers
  • Guide: hero image + step-by-step sections + program providers
  • Library: metadata row + key takeaways + download/ask CTAs
  • Help: urgency badges + service map + grouped resources

4. Implementation Roadmap

Phase 1: Foundation (This Sprint)

  • Build UnifiedDetailLayout (3-column shell)
  • Build LeftNav component with archetypes
  • Refactor DetailWayfinder to match template exactly
  • Build CompactHero with content-type variants
  • Wire into /content/[id] as first page

Phase 2: Roll Out (Next Sprint)

  • Apply to /officials/[id], /services/[id], /organizations/[id]
  • Apply to /policies/[id], /guides/[slug]
  • Apply to /library/doc/[id], /help/[slug]
  • Mobile responsive: drawer nav + stacked wayfinder

Phase 3: Archetype Intelligence

  • Archetype selection persists in cookie (like language)
  • Content ordering in Wayfinder adapts to archetype
  • Homepage PersonaSelector maps to archetypes
  • Analytics: track archetype-driven engagement

Phase 4: Community Features

  • Archetype-specific onboarding flows
  • "Others like you also explored..." recommendations
  • Community contributions surfaced by archetype
  • Partner archetype connects to partner portal

5. What This Solves

  • Consistency: Every page feels like the same platform. No more bespoke layouts.
  • Discoverability: The Wayfinder sidebar ensures every detail page is a doorway to 3 more related things. No dead ends.
  • Intent clarity: Archetypes answer "what can I do here?" before the user even asks.
  • Asset-based framing: The archetype names (Seeker, Learner, Builder, Watchdog, Partner, Explorer) are all empowering identities, not needs-based labels.
  • Scalability: One template handles all 8 content types. New types just need a main-content variant.
